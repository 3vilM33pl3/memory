# Windows

Memory Layer publishes native Windows x86_64 release artifacts:

- `memory-layer-<version>-windows-x86_64.msi`
- `memory-layer-<version>-windows-x86_64.zip`

Windows ARM64 is not a release target yet.

## Native MSI

Download the MSI from GitHub Releases and run it. The installer places
`memory.exe`, the bundled web UI, skill templates, README, example config, and
PowerShell completion under `Program Files\Memory Layer`.

After installing:

```powershell
memory --version
memory wizard --global
memory doctor
memory health
memory tui
```

The native package does not include PostgreSQL. Use an existing PostgreSQL
server with pgvector, a local database you manage yourself, or the bundled
Docker Compose stack.

## Portable ZIP

Use the ZIP when you want to inspect or run the binary without installing the
MSI. Extract the archive, add its directory to `PATH`, then run:

```powershell
memory --version
memory doctor
```

## Docker Desktop

The bundled Compose stack runs under [Docker Desktop](https://docs.docker.com/desktop/setup/install/windows-install/):

```powershell
git clone https://github.com/3vilM33pl3/memory
cd memory
docker compose up
```

That brings up PostgreSQL with pgvector, the Memory Layer service, and the web
UI at [http://localhost:4040](http://localhost:4040). Load the showcase project
from inside the container:

```powershell
docker compose exec memory memory demo
```

## WSL2

For Linux-native shell behavior, TUI workflows, and file watchers, use an Ubuntu
[WSL2](https://learn.microsoft.com/windows/wsl/install) distribution and follow
the [Linux / Debian install](/docs/install/linux-debian) inside it.

Repositories are reachable from WSL2 under `/mnt/c/...`, though keeping active
repositories inside the WSL2 filesystem is faster and works better with file
watchers.
